# Contributing to DerbyNet

## Reporting Bugs

If you're not sure how something is supposed to behave or work, feel free to get in touch by email.

If you think you've encountered a bug, please open a [GitHub issue](https://github.com/jeffpiazza/derbynet/issues/new) to describe your findings.  A good bug report ideally describes:

- What you did,
- What you expected to observe, and
- What you actually observed.

Any accompanying logs or, if relevant, screen shots, would be appreciated.

## Suggesting Enhancements or New Features

Please open a [GitHub issue](https://github.com/jeffpiazza/derbynet/issues) to discuss what you're after.

- Use a **clear and descriptive title** for the issue to identify the suggestion.
- If it helps to describe your idea, screen shot mock-ups might be appropriate.
- **Explain why this enhancement would be useful** to most DerbyNet users, or, alternatively, how it might be presented without being a distraction for users for whom this feature wouldn't apply.

## Contributing Code

If you have a **small** code fix to contribute, please do submit a [pull request](https://github.com/jeffpiazza/derbynet/pull).

For anything more substantial, **please** first open an [issue in GitHub](https://github.com/jeffpiazza/derbynet/issues) so we can discuss what it is you'd like to accomplish.  Let's agree on the goal before getting into the details of the code.
